# 视频NSFW检测系统技术方案

## 1. 系统架构

### 1.1 前端架构
- **框架**: Next.js 14
- **语言**: TypeScript
- **状态管理**: React useState/useRef
- **UI组件**: 原生React组件
- **样式**: CSS Modules

### 1.2 后端架构
- **API服务**: Node.js + Express
- **视频处理**: FFmpeg
- **NSFW检测**: 机器学习模型
- **存储**: 对象存储服务（如AWS S3）

## 2. 核心功能实现

### 2.1 视频上传
- 前端通过文件输入控件选择视频文件
- 使用FormData API构建上传数据
- 通过Axios发送POST请求到后端API
- 支持视频文件类型验证和大小限制

### 2.2 视频处理
- 后端接收视频文件并存储到临时目录
- 使用FFmpeg提取视频帧（每N秒提取一帧）
- 对提取的帧进行预处理（调整大小、标准化）

### 2.3 NSFW检测
- 使用预训练的NSFW检测模型（如YoloV5或OpenCV-based模型）
- 对每个提取的帧进行检测
- 计算视频的整体NSFW评分
- 生成检测报告

### 2.4 结果展示
- 前端接收检测结果
- 展示检测状态（安全/不安全）
- 显示置信度分数
- 可选：显示检测到的敏感帧预览

## 3. 技术选型

### 3.1 前端技术
- **Next.js**: 提供服务端渲染和API路由
- **React**: 构建用户界面
- **TypeScript**: 类型安全
- **CSS Modules**: 模块化样式

### 3.2 后端技术
- **Node.js**: 服务器运行环境
- **Express**: API框架
- **FFmpeg**: 视频处理
- **TensorFlow.js 或 PyTorch**: 机器学习推理
- **Multer**: 文件上传处理
- **AWS S3**: 视频存储

### 3.3 NSFW检测模型
- **选项1**: 使用开源预训练模型（如nsfwjs）
- **选项2**: 自定义训练模型
- **选项3**: 使用第三方API服务（如Clarifai, Sightengine）
